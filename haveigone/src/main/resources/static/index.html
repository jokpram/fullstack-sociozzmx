Sociozzmx Chat API Documentation

Daftar lengkap endpoint API dan struktur JSON untuk aplikasi Chat & WebRTC.

POST /api/auth/register
Mendaftarkan pengguna baru (Registrasi).
Request Body (JSON): 
{
  "username": "johndoe",
  "password": "secretpassword",
  "fullName": "John Doe"
}
POST /api/auth/login
Masuk ke dalam aplikasi (Login). Mengembalikan data user termasuk status Online.
Request Body (JSON): 
{
  "username": "johndoe",
  "password": "secretpassword"
}
POST /api/users/connect
Melaporkan/menyimpan koneksi pengguna ke database (biasanya dipanggil saat WebSocket mau tersambung). Harus menyertakan header Session-Id.
Header Params: Session-Id: <websocket_session_id>

Request Body (JSON): 
{
  "username": "johndoe",
  "fullName": "John Doe"
}
GET /api/users/connected
Mengambil daftar semua pengguna yang saat ini sedang ONLINE.
Request Body: (Tidak ada / Kosong)
GET /api/chat/history
Mengambil riwayat percakapan antara dua pengguna. Parameter `senderId` dan `recipientId` wajib diisi lewat query string.
Query Params: 
?senderId=johndoe&recipientId=janedoe
Response (JSON Array): 
[
  {
    "id": 1,
    "senderId": "johndoe",
    "recipientId": "janedoe",
    "content": "Halo, Jane!",
    "type": "CHAT",
    "timestamp": "2026-02-24T10:00:00"
  }
]
WS/STOMP ws://localhost:8081/ws
Endpoint utama WebSocket. Terhubung menggunakan client STOMP atau SockJS.
Daftar Topic/Queue:
  • /app/chat.sendMessage : Kirim pesan private
  • /app/chat.publicMessage : Kirim pesan grup global
  • /app/webrtc.signal : Kirim sinyal WebRTC (Video Call & Audio Call)
  • /topic/public : (Subscribe) Terima pesan publik atau notifikasi LEAVE
  • /user/queue/messages : (Subscribe) Terima pesan DM pribadi
  • /user/queue/webrtc : (Subscribe) Terima panggilan WebRTC / Answer
Contoh Payload Chat JSON (/app/chat.sendMessage): 
{
  "senderId": "johndoe",
  "recipientId": "janedoe",
  "content": "Halo, Jane!",
  "type": "CHAT",
  "timestamp": "2026-02-24T10:00:00"
}
WS/STOMP /app/webrtc.signal
Mengirim dan menerima SDP Offers, Answers, serta rentetan protokol ICE Candidate untuk menghubungkan Peer-to-Peer Video Call / Voice Call. Target menerima di /user/queue/webrtc
Contoh Payload Video Call (SDP Offer): 
{
  "type": "offer",
  "sender": "johndoe",
  "recipient": "janedoe",
  "sdp": "v=0\r\no=- 461173... \r\ns=-\r\nt=0 0\r\n..."
}
POST /api/stories
Membuat (Upload) status/story baru yang akan hilang dalam 24 jam. Ini adalah permintaan Multipart FormData jika menggunakan gambar/video.
Request Type: multipart/form-data

Form Fields:
  • userId : String (username pembuat)
  • mediaFile : File biner (.jpg / .mp4)
  • caption : String opsional
